JSON 格式：

JSON 约束：
- 只能输出一个 JSON object，不要输出 Markdown fence、解释文字或前后缀。
- summary 是 JSON 字符串；如需换行，必须用合法 JSON 转义（例如 \n\n），不要在字符串里输出未转义的原始换行。
- tags 输出 2-5 个标签，中文或英文均可，描述主题、技术方向、公司/产品或讨论焦点。

score 字段是“条目综合价值分”，不是单纯关键词相关性。按 1-10 分评价这条新闻是否值得进入技术日报：
- 主题相关性：是否与 AI、LLM、agent、模型、推理训练基础设施、开发者工具、开源生态、监管或产业变化强相关。
- 时效性：是否是近期发布、重要变化、事故、政策、产品更新或社区关注的新事件。
- 意义与影响：是否影响开发者、企业采用、产品路线、成本结构、竞争格局、开源生态或监管环境。
- 创新性：是否有新方法、新架构、新能力、新数据、新结论，而不是旧信息重发。
- 焦点性：是否处于行业或社区争议中心，是否有持续跟踪价值。
- 信息密度：是否包含具体事实、数字、技术细节、约束条件或一手经验。

评分参考：
- 9-10：重大、高时效、高影响，有明确技术/产业意义和持续跟踪价值。
- 7-8：相关且有信息增量，值得收录，但影响范围或创新性有限。
- 5-6：有一定相关性，但主要是渐进更新、转述、弱信息或局部事件。
- 3-4：只沾 AI 关键词，主体是消费硬件、导购、促销、营销包装、泛泛报道或重复信息。

summary 字段必须包含以下四节 Markdown 文本，节标题必须原样保留：

【短总结】
用约 200 字概括整条新闻，必须覆盖 5W+1H：谁（Who）、何时（When）、何地/场景（Where，如适用）、发生了什么（What）、为什么重要（Why）、如何实现或如何影响（How）。这一段要先给结论，再给关键事实和影响判断；不要复述标题，不要写“本文介绍了”。

【详细说明】
用约 1000 字做深入说明，信息不足时宁可明确“不足以判断”，不要编造。必须覆盖：
- 5W+1H 展开：主体、时间、场景、事件、原因/动机、机制/路径。
- 关键事实：产品、公司、人物、模型、参数、价格、性能、规模、时间线等具体信息；原文没有数字就说明未披露。
- 技术与产品分析：它解决什么问题，采用什么方法，和已有方案相比有什么差异。
- 意义与影响：对开发者、企业、用户、产业竞争、开源生态或监管的影响。
- 创新性与局限：哪些内容是真正的新变化，哪些只是包装、营销或渐进更新；指出不确定性和需要继续观察的点。
- 价值判断：说明这条新闻为什么值得或不值得关注，避免只做信息搬运。
